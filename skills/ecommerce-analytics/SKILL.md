# 电商数据诊断引擎

本 skill 的核心使命：**给我数据，我诊断问题根因，我给可执行的优化建议。**

不是列指标，不是做报表，而是像医生读化验单一样 — 看到异常数字，往下追溯，找到病因，开出处方。

## 工作流程

### 第一步：接收并结构化数据

用户可能以任何格式提供数据：
- 粘贴的表格数据
- GA4 截图中的数字
- 广告后台数据
- Shopify 后台导出的 CSV
- 口头描述的几个关键数字
- 混合格式（部分截图 + 部分数值）

**处理方式**:
1. 从数据中提取所有可用数字，建立数据结构
2. 识别数据类型：流量数据？漏斗数据？广告数据？销售数据？混合？
3. 识别时间维度：是否有对比期（同比/环比）？时间粒度？
4. 如果数据不足以诊断，明确告知需要什么补充数据

### 第二步：异常检测

对数据进行系统性扫描，找出所有异常信号：

- **偏离基准**: 哪些指标明显偏离行业基准或历史均值？
- **突变点**: 哪个指标在某时间点突然变化？
- **结构性异常**: 指标间的关系是否合理？（如流量涨了但转化没涨）
- **异常低值**: 哪个环节的数值异常低？
- **异常高值**: 哪个环节的数值异常高？（高值也可能是问题，如异常高的加购率可能是爬虫）

### 第三步：根因分析

发现异常后，向下追溯根本原因。关键原则：**不停在表面现象，至少再深挖两层。**

例如：
- 表面：转化率从 2.5% 降到 1.8%
- 第一层：是哪个渠道转化率掉了？
- 第二层：该渠道的落地页跳出率是否变了？还是加购到结算的流失增加了？
- 第三层：如果是跳出率变了，是流量质量变了还是页面出了问题？

### 第四步：诊断报告输出

### 第五步：优化建议

## 诊断维度

### 1. 漏斗诊断

电商漏斗的核心环节和关键指标：

```
访问 → 产品页浏览 → 加购 → 进入结算 → 支付成功
 |        |            |         |           |
Sessions  PDP Views   Add-to-  Checkouts   Purchases
                       Cart
 |        |            |         |           |
 ─────→ 转化率1 →──── 转化率2 →── 转化率3 →── 转化率4
```

| 漏斗环节 | 关键指标 | 基准参考（DTC） | 异常信号 |
|----------|----------|-----------------|----------|
| 访问→PDP | PDP 浏览率 | 40-60% 访客至少看1个PDP | 低于30%说明流量不匹配或导航有问题 |
| PDP→加购 | 加购率 | 8-15% | 低于5%需检查PDP质量/价格/库存 |
| 加购→结算 | 结算率 | 40-60% | 低于30%说明结算门槛高或信任不足 |
| 结算→支付 | 支付完成率 | 60-80% | 低于50%需检查支付流程/运费透明度 |
| 整体 | 站内转化率 | 1.5-3% | 低于1%需全面诊断 |

*基准仅为参考，不同品类/客单价差异大，优先和自己历史数据对比*

**漏斗诊断方法**:
1. 计算每个环节的转化率和流失率
2. 找到流失最大的环节（这是优先修复点）
3. 与历史数据对比，看哪个环节恶化
4. 分析该环节流失的可能原因

### 2. 流量质量诊断

| 指标 | 说明 | 异常判断 |
|------|------|----------|
| 各渠道流量占比 | 流量结构是否健康 | 单一渠道占比>70%有风险 |
| 各渠道转化率对比 | 哪个渠道质量最高 | 转化率差距>3倍需关注 |
| 跳出率 | 流量与落地页匹配度 | 整体>70%或某渠道>80% |
| 新访 vs 回访比例 | 用户结构 | 回访<10%说明复购/留存差 |
| 平均会话时长 | 用户参与度 | <30秒说明流量质量差 |
| 页面/会话 | 用户深度 | <2页说明没有引导浏览 |

**流量质量诊断方法**:
1. 按渠道拆分流量和转化数据
2. 比较各渠道的转化效率（CVR、ROAS）
3. 检查是否有异常流量（高流量零转化 = 无效流量）
4. 分析流量结构变化（某个渠道突然涨/跌）

### 3. 广告效率诊断

| 指标 | 说明 | 诊断要点 |
|------|------|----------|
| ROAS | 广告回报率 | 低于2倍覆盖不了运营成本（视利润率而定） |
| MER | 整体营销效率比 | 总收入/总广告支出，比ROAS更全面 |
| CPA | 单次获客成本 | 高于客户首单利润则亏损 |
| CTR | 点击率 | 低于1%说明创意/受众不匹配 |
| CPC | 单次点击成本 | 异常升高说明竞争加剧或受众质量下降 |
| CPM | 千次展示成本 | 突然上升可能说明受众过窄或竞价激烈 |
| 频率 | 广告展示频率 | >3次创意疲劳，需要换素材 |
| 花费速度 | 预算消耗速度 | 花不出去=受众/竞价问题，花太快=可能浪费 |

**广告效率诊断方法**:
1. 先看整体 MER：总收入/总广告支出，判断整体效率
2. 按广告系列/广告组拆分 ROAS
3. 交叉验证：CTR 低 + CPC 高 = 创意差 + 受众竞争大
4. 区分冷流量/暖流量/热流量的 ROAS 差异
5. 检查广告支出和收入的时滞关系（广告今天花，收入可能7天后才来）

### 4. 产品表现诊断

| 指标 | 说明 | 诊断要点 |
|------|------|----------|
| 畅销品 Top 10 | 头部产品贡献 | Top 10 占收入>50%有集中风险 |
| 滞销品 | 零动销产品 | 库存积压，需要清仓或下架 |
| 退货率 | 退货/订单比 | >10%需检查产品质量/描述准确性 |
| 客单价 (AOV) | 订单均价 | 突然下降可能因为低客单品销量增加 |
| 产品加购率 | 加购/PDP浏览 | 低于5%说明产品页说服力不足 |
| 产品转化率 | 购买/PDP浏览 | 与加购率差距大说明结算环节有问题 |

### 5. 用户行为诊断

| 指标 | 说明 | 诊断要点 |
|------|------|----------|
| 新客 vs 回购比例 | 客户结构 | 新客>90%说明留存差 |
| 复购率 | 下单2次以上客户占比 | 低于15%需改善留存（取决于品类） |
| AOV 变化趋势 | 客单价方向 | 下降趋势需通过捆绑/推荐拉升 |
| 客户生命周期价值 (LTV) | 客户长期贡献 | LTV/CAC < 3 说明获客成本高或留存差 |
| 首单到复购间隔 | 多久复购 | 间隔过长说明触达不足 |
| 邮件/短信互动率 | 留存渠道效果 | 打开率<15%或点击率<1%需优化 |

## 核心指标定义和关系

### 基础指标

- **AOV** (Average Order Value) = 总收入 / 订单数
- **CVR** (Conversion Rate) = 订单数 / 访问数 × 100%
- **CAC** (Customer Acquisition Cost) = 总获客支出 / 新客户数
- **LTV** (Lifetime Value) = 客户平均订单数 × AOV × 毛利率
- **ROAS** (Return on Ad Spend) = 广告带来收入 / 广告支出
- **MER** (Marketing Efficiency Ratio) = 总收入 / 总广告支出
- **CPA** (Cost Per Acquisition) = 广告支出 / 转化次数
- **CAC:LTV 比** = LTV / CAC，健康值 > 3

### 指标间的因果关系

```
广告支出 → 流量 → PDP浏览 → 加购 → 结算 → 支付
   |          |        |        |       |       |
  CPC       跳出率   加购率   结算率  支付率   AOV
   |          |        |        |       |       |
  CTR       流量质量  PDP质量  信任度  支付体验  产品定价
   ↓          ↓        ↓        ↓       ↓       ↓
  ROAS ← ← ← ← ← 最终收入 ← ← ← ← ←
```

理解这些关系才能做根因分析，而不是孤立看单个指标。

## 根因分析方法论

### 追问三层面

当发现一个异常指标时，追问三个层面：

1. **量层面**：数字变了吗？变了多少？什么时候变的？
2. **结构层面**：是整体变了还是某个部分变了？（某个渠道？某个产品？某个市场？）
3. **原因层面**：为什么会变？（外部因素？内部改动？竞争对手？季节性？）

### 常见根因模式

| 表面现象 | 可能根因 | 验证方法 |
|----------|----------|----------|
| 整体转化率下降 | 某个低质量渠道流量占比增加 | 按渠道拆分转化率，排除该渠道后看趋势 |
| 加购率正常但支付率低 | 运费/税费在结算页才显示，吓跑客户 | 检查结算页跳出率和运费可见性 |
| ROAS 下降 | CPC 上涨（竞价加剧）或 CVR 下降 | 拆分看 CPC 和 CVR 各自趋势 |
| 流量增长但收入不变 | 流量质量下降（垃圾流量/不匹配受众） | 看跳出率、页面/会话、渠道来源 |
| AVO 下降 | 低客单产品销量增加或折扣比例增加 | 按产品拆分收入贡献 |
| 复购率下降 | 产品质量问题/竞品活动/触达不足 | 查看退货率、客户反馈、邮件互动率 |
| 支付完成率低 | 支付方式不够/支付流程出错/运费过高 | 按支付方式拆分成功率，检查结算页 |
| 购物车放弃率高 | 运费不透明/必须注册/结算步骤多 | 检查结算流程步骤数和运费显示位置 |

### 排除法

当多个可能原因并存时，用排除法缩小范围：

1. **时间匹配**：异常出现的时间点，是否匹配某个改动（上线新页面、改广告素材、调价格）？
2. **渠道隔离**：如果只有某个渠道转化率下降，问题在该渠道而非网站
3. **设备对比**：如果只有移动端转化率下降，问题在移动端体验
4. **新旧客对比**：如果只有新客转化率下降，问题在流量质量；如果老客也下降，问题在网站/产品

## 诊断报告格式

ALWAYS 使用以下格式输出诊断结果：

```markdown
# 数据诊断报告

## 数据概览
- 时间范围：[日期]
- 对比基准：[历史同期/上周/上月]
- 关键指标汇总（数字 + 变化幅度）

## 发现的问题（按影响程度排序）

### 问题 1：[问题标题]
- **数据证据**：[具体数字和变化]
- **可能原因**：[分析原因]
- **验证方法**：[如何确认这个原因]
- **优化建议**：[具体可执行的动作]
- **预期影响**：[修复后可能改善多少]

### 问题 2：...

## 优先行动清单

| 优先级 | 行动 | 预期效果 | 所需资源 |
|--------|------|----------|----------|
| P0 | ... | ... | ... |
| P1 | ... | ... | ... |
| P2 | ... | ... | ... |

## 需要补充的数据
[如果数据不足以完整诊断，列出还需要什么数据]
```

## 红线原则

1. **数据不足时必须声明**，不要硬分析。明确说"当前数据不足以诊断 X 问题，需要补充 Y 数据"
2. **区分相关性和因果性**。两个指标同时变化不代表因果，必须给出推理链
3. **不要给模糊建议**。"优化落地页"不是建议。"落地页跳出率 85%，主要因为首屏加载时间 4.2 秒，建议压缩图片到 WebP 格式将加载时间降到 2 秒以内"才是建议
4. **每个建议必须可执行**。具体到做什么、怎么做、预期什么效果
5. **不要忽略正面数据**。问题要诊断，优势也要指出 — 哪里做得好可以加大投入
6. **基准值只是参考**。不同品类、客单价、市场的基准差异大，优先和自己的历史数据对比
7. **不要只看单点数据**。一个数字没有意义，必须有对比（时间对比、渠道对比、产品对比）

## 常见数据陷阱

| 陷阱 | 说明 | 正确做法 |
|------|------|----------|
| 平均值掩盖分布 | 整体转化率 2%可能由1%渠道A和5%渠道B构成 | 始终拆分看分布 |
| 小样本误判 | 10个访客2个转化=20%转化率，不具统计意义 | 确保样本量足够大再做判断 |
| 季节性误解 | 11月销售涨了可能只是黑五效应 | 与去年同期对比，不只是环比 |
| 归因错误 | 把收入都归给最后一次点击，忽略种草渠道 | 了解归因模型局限性 |
| 虚荣指标 | 页面浏览量涨了但收入没涨 | 关注和收入直接相关的指标 |
| 幸存者偏差 | 只分析已下单客户，忽略放弃购物车的客户 | 分析流失用户的行为 |
| 辛普森悖论 | 每个渠道转化率提升但整体下降 | 检查渠道结构变化 |
