# 飞书费用台账录入

从费用截图提取信息，校验白名单，用户确认后写入飞书 Base 多维表格。

## 工作流程

```
截图 → 提取(金额/日期/服务商/费用名) → 查白名单 → 用户确认 → 写入飞书
```

### 第一步：读取用户白名单

每个用户有各自的费用类型白名单。先读 `references/user-whitelists.md` 找到当前用户的配置。

如果没有找到当前用户，先让用户定义他需要报销的费用类型，然后写入 `references/user-whitelists.md`，再继续。

### 第二步：从截图提取信息

从用户提供的截图中提取以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| **金额** | 扣款金额，保留小数 | $29.00、¥199.00 |
| **日期** | 扣款发生日期 | 2026-05-15 |
| **服务商** | 收款方/服务名称 | Shopify、Cursor、Google One |
| **费用名称** | 具体费用项目 | Shopify Basic Plan 月费、Cursor Pro 订阅 |

提取规则：
- 金额只取数字，去掉货币符号前后的空格
- 日期统一为 YYYY-MM-DD 格式
- 服务商名称取官方全称（如 "Google One" 而非 "GOOGLE"）
- 费用名称尽量具体（包含 Plan 名称、订阅周期等）

**如果截图信息不全**（日期模糊、金额看不清、服务商不明确），必须先问用户补充，不要猜测填写。

### 第三步：白名单校验

将提取的**服务商**与当前用户的白名单对比：

- **在白名单内** → 进入第四步，准备写入
- **不在白名单内** → 告知用户该服务商不在报销范围，询问是否需要录入（如果是新类型，可更新白名单）
- **匹配不确定** → 列出可能匹配的白名单项，让用户确认

### 第四步：用户确认

写入飞书前，必须把提取内容展示给用户确认：

```
确认录入以下费用：

| 字段 | 内容 |
|------|------|
| 金额 | $29.00 |
| 日期 | 2026-05-15 |
| 服务商 | Shopify |
| 费用名称 | Shopify Basic Plan 月费 |

确认无误？回复"确认"即写入飞书。
```

用户确认后才能执行写入。如果用户要求修改任何字段，修改后重新展示确认。

### 第五步：写入飞书 Base

确认后，用 Python 内联脚本直接调用飞书 API（比走外部脚本更稳定）：

```python
# 写入时会自动：
# 1. 从环境变量读取 FEISHU_APP_ID / FEISHU_APP_SECRET / FEISHU_BITABLE_APP_TOKEN / FEISHU_TABLE_ID
# 2. 获取 tenant_access_token
# 3. 调用 batch_create API（单条接口在某些工作空间会 403）
# 4. 日期自动转为 Unix 毫秒时间戳（UTC+8）
```

**表格字段映射**（以 📇费用台账 实际字段为准）：

| 提取字段 | 表格字段 | 类型 | 格式说明 |
|---------|---------|------|---------|
| 服务商 + 费用名 | 报销摘要 | 文本 | `{服务商} - {费用名称}` |
| 金额 | 费用金额 | 数字 | 纯数字，不含货币符号 |
| 日期 | 报销日期 | 日期 | Unix 毫秒时间戳（UTC+8） |
| 类别 | 报销科目 | 单选 | 默认「办公」，可配 AI费用/店铺订阅套餐 |
| — | 报销进度 | 单选 | 默认「未报销」 |
| 补充信息 | 备注 | 文本 | 服务商等额外信息 |

**注意**：
- 首次使用需要配置飞书应用凭证，按 `references/feishu-api.md` 指引操作
- 必须使用 **batch_create** 接口，单条创建接口可能返回 403
- 日期必须是 Unix 毫秒时间戳，不能用字符串
- 如果脚本报错缺少配置，按 `references/feishu-api.md` 帮助用户完成配置

## 重要规则

- **不猜测** — 信息不全就问，不要假设
- **不跳过确认** — 写入飞书前必须用户确认
- **不修改白名单** — 除非用户明确要求添加新的费用类型
- **统一货币** — 保持截图中的原始货币，不要转换
- **一条一录** — 如果截图包含多条费用，逐条确认逐条录入

## 文件结构

- `references/user-whitelists.md` — 用户白名单配置
- `references/feishu-api.md` — 飞书 API 配置与接口参考
- `scripts/write_record.py` — 写入飞书的可复用脚本
